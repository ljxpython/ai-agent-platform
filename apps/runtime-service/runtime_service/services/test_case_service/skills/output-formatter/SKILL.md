# 输出格式化 Skill

## 激活场景
- quality-review 完成后，作为最后一个阶段输出最终结果
- 用户要求「导出测试用例」「格式化输出」「生成测试报告」
- 用户指定特定格式（Markdown / JSON / CSV）
- 需要将测试用例整理为可分享的文档

## 支持的输出格式

### 格式 A：Markdown 完整报告（默认）
最完整的输出格式，包含所有分析阶段的产出。

```markdown
# 测试用例分析报告

**项目**：[项目名称]
**生成时间**：[时间]
**总用例数**：[N]
**质量评分**：[分数]/100

---

## 一、需求分析摘要
[requirement-analysis 产出]

## 二、测试策略
[test-strategy 产出]

## 三、测试用例集

### 模块：[模块名]

| 用例编号 | 用例标题 | 优先级 | 类型 | 预期结果 |
|---------|---------|--------|------|----------|
| TC-XXX-001 | ... | P0 | 功能 | ... |

#### TC-XXX-001 详细信息
**前置条件**：...
**测试步骤**：
1. ...
**测试数据**：...
**预期结果**：...

## 四、质量评审报告
[quality-review 产出]
```

### 格式 B：JSON 结构化数据
适用于程序化处理和工具集成。

```json
{
  "report": {
    "generated_at": "ISO8601时间",
    "total_cases": 0,
    "quality_score": 0
  },
  "test_cases": [
    {
      "id": "TC-XXX-001",
      "title": "用例标题",
      "priority": "P0",
      "type": "功能",
      "design_technique": "等价类",
      "module": "模块名",
      "preconditions": ["前置条件"],
      "steps": ["步骤1", "步骤2"],
      "test_data": {"字段": "值"},
      "expected_result": "预期结果",
      "notes": "备注"
    }
  ]
}
```

### 格式 C：CSV（Excel友好）
适用于导入禅道、JIRA、TestRail 等测试管理工具。

```
用例编号,用例标题,模块,优先级,测试类型,前置条件,步骤1,步骤2,步骤3,测试数据,预期结果,备注
TC-XXX-001,用例标题,模块名,P0,功能,前置条件,步骤1,步骤2,,数据,预期结果,
```

## 执行流程

### Step 1：收集所有阶段产出
从文件系统（虚拟或持久化）收集各阶段产出文件：
- `requirement_analysis.md`
- `test_strategy.md`
- `test_cases.md` / `test_cases.json`
- `quality_review.md`

### Step 2：确定输出格式
根据用户指定或默认规则选择格式：
- 未指定 → Markdown 完整报告
- 指定「JSON」→ 格式 B
- 指定「CSV」或「Excel」→ 格式 C
- 指定「简洁」→ 仅用例集部分

### Step 3：数据整合与格式化
- 去重：相同编号的用例保留最新版本
- 排序：按模块 → 优先级（P0→P3）→ 编号排序
- 统计：输出用例数量汇总、各优先级占比

### Step 4：统计摘要生成

```markdown
## 测试用例统计

| 指标 | 数值 |
|------|------|
| 总用例数 | N |
| P0用例 | N（X%）|
| P1用例 | N（X%）|
| P2用例 | N（X%）|
| P3用例 | N（X%）|
| 正向用例 | N（X%）|
| 负向用例 | N（X%）|
| 边界用例 | N（X%）|
| 质量评分 | X/100 |
```

### Step 5：最终输出
将格式化后的内容作为最终 assistant 消息输出。

## 输出规范

1. **📊 统计摘要**：用例数量、优先级分布、质量评分
2. **📋 格式化用例集**：按选定格式完整输出
3. **🔗 工具导入指南**：如何将输出导入常用测试工具（如有需要）
4. **✅ 完成确认**：明确标注分析完成，用户可以直接使用

## 关键约束

> 🎯 **完整性优先**：宁可输出略长，也要确保所有用例都被包含在输出中。
> 🔢 **编号连续**：确保输出中用例编号连续，无跳号。
> 📌 **格式一致**：同一格式内所有用例的字段顺序和格式必须一致。
> 🚫 **禁止截断**：不能以「更多用例请见附件」等方式截断输出，必须完整输出。
