# 输出格式化 Skill

## 激活场景
- `quality-review` 已完成
- 需要对外输出最终结果
- 需要在输出后立刻持久化

## 默认输出格式（紧凑模式）
除非用户明确要求完整报告，否则默认输出以下四段：
1. `需求摘要`
2. `正式测试用例`
3. `质量结论`
4. `持久化结果`（仅在工具调用后填写）

## 格式化要求
- 保留正式测试用例的结构化字段
- 文本简洁，避免长背景说明
- 先整理出可传给工具的 `bundle_title`、`bundle_summary`、`test_cases`、`quality_review`

## 持久化联动规则
- 如果任务要求“保存/落库/入库/持久化”，格式化完成后不要停留在长篇展示
- 应立即进入 `test-case-persistence`，再调用 `persist_test_case_results`
- 不要为了展示完整报告而阻塞落库

## 约束
- 完整长报告只有在用户明确要求时才输出
- 输出必须优先服务于工具调用和最终交付，而不是追求篇幅
