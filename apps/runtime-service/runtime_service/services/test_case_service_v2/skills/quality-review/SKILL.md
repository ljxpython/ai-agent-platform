# 质量评审 Skill

## 激活场景
- `test-case-design` 已完成
- 需要给出质量结论或交付前自检

## 必做动作
1. 检查核心覆盖是否完整
2. 检查步骤和预期结果是否可执行、可验证
3. 识别阻断问题和重要风险

## 紧凑模式（默认）
默认只输出：
- `quality_score`：0-100
- `quality_gate`：`pass` / `needs_fix`
- `summary`：1 句评审结论
- `major_risks`：最多 3 条问题或风险

## 门禁判断
- 主流程缺失、预期结果不可验证、步骤无法执行：判定 `needs_fix`
- 无阻断问题时，可判定 `pass`

## 约束
- 只指出关键问题，不要写成长篇评分报告
- 如果发现阻断问题，应先修正用例内容，再进入格式化或持久化阶段
