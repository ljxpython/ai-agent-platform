# Platform Web 控制面页面开发标准

这份文档把 `apps/platform-web` 的正式页面范式固定下来。后续页面继续开发，不再回到“先把页面拼出来再说”的散装模式。

视觉壳层规范单独收敛在：

- `docs/frontend-visual-baseline-standard.md`

---

## 1. 页面结构

每个正式页面默认拆成 4 层：

1. 路由入口
2. 页面壳层
3. service 调用层
4. 复用组件层

固定要求：

- 路由只做参数承接和鉴权进入
- 页面只负责编排展示与交互
- 请求统一从 `src/services/*` 发起
- 复用组件不直接依赖具体业务接口

---

## 2. UI 标准

- 页面必须有稳定标题区
- 导航、顶部条、标题区、卡片、筛选、表格、弹层必须优先复用共享壳
- 说明块、批次上下文块、详情弹窗块、聊天检查面板统一使用 `pw-panel*` 系列
- 列表页优先使用统一筛选区、统计卡、表格区三段结构
- 风险动作必须有确认弹窗
- loading / empty / error 不能省
- 大屏下内容区必须撑满，不允许出现残缺留白
- 深色模式和浅色模式都要保持首屏一致性
- 禁止页面继续扩散散装的玻璃态、重阴影、大圆角壳

---

## 3. 交互标准

- 不自动做让用户迷路的跳转
- 长列表默认保留筛选和分页状态
- 后台轮询或流式更新不能强行抢滚动焦点
- Drawer / Modal 打开后要保证小分辨率可见
- 需要用户确认的动作，不允许只靠回车误触发
- 顶栏和侧栏的视觉口径不能被业务页覆盖，业务页只消费，不改写壳层

---

## 4. Service 标准

- 页面不直接拼接 URL
- 模块请求统一收敛到对应 service
- service 负责接口契约归一化
- 错误解析统一走共享 HTTP 错误处理
- 不允许把 legacy fallback 写回正式 service
- 正式页面统一只走 `platform-api`
- 正式 service / client 命名不再使用 `legacy*`、`platformV2*` 这类过渡期前缀
- 项目上下文统一通过 `useWorkspaceProjectContext`
- 不再维护 `runtimeProjectId / runtimeScopedProject / runtimeProjects` 这类伪独立上下文别名
- 页面不要再自行判断 `resolvePlatformClientScope(...) === 'v2'`
- 长耗时导出、刷新、重同步优先走 `operations`

---

## 5. State 标准

- 全局登录态放 store
- 当前项目上下文放 store
- 页面筛选、弹窗、临时表单状态优先放页面内部
- 不把一次性弹窗状态塞进全局 store
- 不维护双 token、双项目上下文、双控制面状态
- `workspace` store 只承载唯一正式项目上下文
- 过渡期别名只能作为兼容层存在，新增页面禁止继续扩散

---

## 6. 权限接入标准

- 路由级访问控制统一放在路由 `meta.requiredPermissions`
- 页面内动作统一通过 `useAuthorization()` 判断
- 导航是否展示、按钮是否可点、危险动作是否可执行，必须使用同一套权限语义
- 读权限页面里如果存在刷新、编辑、删除、重同步等写动作，必须再做按钮级 permission gate
- 不允许在页面里继续手写 `is_super_admin`、`admin/editor/executor` 之类的散装判定
- 项目级权限判断必须带 `projectId`，不能偷懒省略
- 无权限时优先表现为导航裁剪、按钮禁用、友好提示，真实安全仍由后端兜底

---

## 7. 审计标准

前端虽然不直接写审计，但正式页面必须遵守审计触发点：

- 创建
- 更新
- 删除
- 启停
- 批量操作
- 配置变更

页面开发时至少要能回答：

- 这个动作命中哪个后端接口
- 后端会产出哪个 audit action
- 成功和失败后用户能在哪里看到结果

---

## 8. 新页面最低交付物

- 页面源码
- 对应 service
- 类型定义
- 最小 loading / empty / error
- 手动验收步骤
- 必要时补路由和导航入口

---

## 9. 明确禁止

- 页面直接发裸请求
- 页面偷偷拼 legacy 地址
- 一个页面同时维护两套控制面数据源
- 把调试性质组件直接塞进正式导航
- 为了赶进度省掉空态、错态、确认弹窗
- 继续新增“判断 scope 后走不同接口”的过渡期分支
- 页面直接手写新的布局壳和视觉壳，不复用共享样式
