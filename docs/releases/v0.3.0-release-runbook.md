# v0.3.0 发版执行清单

## 1. 目标

这份清单用于准备 `v0.3.0` 发版，固定“平台正式宿主命名收口版”的执行顺序。

当前默认口径：

- 版本号：`v0.3.0`
- 发版对象：整个仓库
- 发版基线：`main`
- 版本口径：`平台正式宿主命名收口版`

## 2. 参考文档

发版前先看：

- [release-policy.md](./release-policy.md)
- [v0.3.0-agent-workspace-demo-draft.md](./v0.3.0-agent-workspace-demo-draft.md)
- [CHANGELOG.md](../CHANGELOG.md)
- [docs/local-deployment-contract.yaml](../local-deployment-contract.yaml)

## 3. 发版前准备

至少确认：

- 本轮 canonical rename 已合并到 `main`
- `docs/CHANGELOG.md` 已形成 `v0.3.0` 正式条目
- `v0.3.0-agent-workspace-demo-draft.md` 已可直接用于 GitHub Release
- 活动面 residue scan 已清空旧正式宿主命名
- 本地脚本 / Docker stack / current docs 已统一到 `platform-api` / `platform-web`

## 4. 最终校验

```bash
uv run --project apps/platform-api python -m unittest discover -s tests -v
uv run --project apps/platform-api python -m compileall app
pnpm --dir apps/platform-web test:run
pnpm --dir apps/platform-web check
docker compose -f deploy/docker-compose.stack.yml --env-file deploy/.env.stack config
docker compose -f deploy/docker-compose.stack.nginx.yml --env-file deploy/.env.stack config
bash ./scripts/platform-web-demo-up.sh
bash ./scripts/platform-web-demo-health.sh
bash ./scripts/platform-web-demo-down.sh
```

## 5. 打 tag

```bash
git checkout main
git pull origin main
git tag -a v0.3.0 -m "release: v0.3.0"
```

## 6. 推送 main 与 tag

```bash
git push origin main
git push origin v0.3.0
```

## 7. 创建 GitHub Release

如使用 `gh`：

```bash
gh release create v0.3.0 \
  --title "v0.3.0 - 平台正式宿主命名收口版" \
  --notes-file "docs/releases/v0.3.0-agent-workspace-demo-draft.md"
```

如需先人工确认，也可以先加 `--draft`。

## 8. 发版后确认

- 远端 tag 已存在
- GitHub Release 已创建
- `docs/CHANGELOG.md` 的 `v0.3.0` 条目与 Release 文案一致
- `Unreleased` 已切回下一版本
