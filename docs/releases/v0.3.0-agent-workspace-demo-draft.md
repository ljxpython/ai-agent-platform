# v0.3.0 - 平台正式宿主命名收口版

> 正式发布说明草案，可直接用于 GitHub Release。

## Summary

`v0.3.0` 是 `v0.2.0` 之后的一轮仓库级收口版本。
这一版没有继续扩展业务功能，而是把已经成为当前事实的正式平台宿主命名彻底收口：

- `apps/platform-api-v2` → `apps/platform-api`
- `apps/platform-web-vue` → `apps/platform-web`

核心结果是：

- 当前正式平台链路在代码、部署、脚本、CI 与 current docs 中不再混用过渡期命名
- operator、开发者、测试与 AI 代理面对的是同一套正式宿主口径
- 本地 demo、Docker stack、current-standard 与 leaf docs 的主链说明重新对齐

## Highlights

- 正式控制面宿主统一为 `platform-api`
- 正式平台前端宿主统一为 `platform-web`
- CI、Docker Compose、Nginx、helper scripts 与 env contract 全面对齐新名称
- runtime / interaction-data / runtime-web 等当前文档中的平台主链口径同步收口
- 活动面已无 `platform-api-v2` / `platform-web-vue` 残留（历史桶除外）

## Added

- 新增 `v0.3.0` 发版草案与 runbook

## Changed

- 将 `apps/platform-api-v2` 整体迁移为 `apps/platform-api`
- 将 `apps/platform-web-vue` 整体迁移为 `apps/platform-web`
- 将仓库级 current-standard、默认本地部署 contract、README/README.en、leaf docs、CI workflow、docker compose、nginx upstream、helper scripts 收口到 `platform-api` / `platform-web`
- 将前端与后端的当前 env / package / path identity 对齐到新的正式宿主名称

## Fixed

- 修复当前活动面里旧正式宿主命名残留导致的 CI 路径、helper scripts、deploy service 名与 current docs 漂移
- 修复 `platform-api-v2` / `platform-web-vue` 与当前真实主链不一致造成的 operator 认知分裂
- 修复 runtime / interaction-data / runtime-web 当前文档中仍把旧名称当作正式链路入口的问题

## Known Issues

- `apps/platform-web` 当前 lint 仍有仓库既有 warnings，但无 error，不阻塞当前版本发布
- 旧名称仍保留在历史桶中（如 release/archive/migration-history/.omx），这是有意保留的历史语境，不代表当前正式宿主
- `omx_state` MCP 在会话尾部有 transport closed 现象，但不影响本次代码与验证结果

## Recommended Demo Flow

建议对外按下面顺序说明：

1. 当前正式平台宿主命名已经统一为 `platform-web` / `platform-api`
2. 展示 `docs/local-deployment-contract.yaml`
3. 展示根脚本：
   - `scripts/dev-up.sh`
   - `scripts/check-health.sh`
   - `scripts/dev-down.sh`
4. 展示本地平台链路：
   - `platform-web -> platform-api -> runtime-service`
5. 展示 Docker stack 入口
6. 展示 README / README.en / current-standard 已统一到新名称
