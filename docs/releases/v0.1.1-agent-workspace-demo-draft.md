# v0.1.1 - Agent 工作台可演示版

> 正式发布说明草案，可直接用于 GitHub Release。

## Summary

`v0.1.1` 是 `v0.1.0` 之后的一轮稳定性补强与主线收口版本。
这一版把正式控制面切换到 `platform-api-v2`，进一步补齐 `platform-web-vue` 的工作台主线、统一 `runtime-service` 的 harness 范式，并把根级与应用级文档重新对齐到当前正式架构。

## Highlights

- `platform-api-v2` 正式承接新的平台控制面主线，补齐 runtime gateway、治理模块与 testcase 管理能力
- `platform-web-vue` 继续收口为正式工作台宿主，统一品牌、导航、上下文切换与 chat / sql-agent / testcase / threads 演示主线
- `runtime-service` 完成首批静态 graph 标准化，沉淀 assistant / deepagent / testcase 的 harness 开发范式
- 根级文档、app docs 和正式架构图已统一到 `AI Harness` 总哲学，不再把历史宿主误写成当前实现

## Added

- 新增 `platform-api-v2` 应用与相关治理/网关模块
- 新增 `runtime-service` harness scaffold、开发范式说明与标准化 graph 主线
- 新增 `platform-web-vue` 的更多正式工作台页面能力与系统治理入口

## Changed

- 收口 `platform-web-vue` 的品牌口径、侧边栏结构、顶栏系统区、项目上下文与运行时目录体验
- 将历史 `platform-web` / `platform-api` 主线降级为兼容或归档语境，正式入口切换到 `platform-web-vue` / `platform-api-v2`
- 重写根级文档、应用文档与正式架构图，统一仓库叙事、默认链路与结果域边界

## Fixed

- 修复 chat 中的会话过期跳转、图表工具结果渲染与若干阅读体验问题
- 修复流式运行时契约对齐问题，稳定 chat / runtime 联调主线
- 修复文档对历史控制面和历史前端宿主的误导性当前态表述

## Verification

- `pnpm install --frozen-lockfile` (`apps/platform-web-vue`)
- `pnpm lint` (`apps/platform-web-vue`)
- `pnpm build` (`apps/platform-web-vue`)
- 根级 / app docs 目标范围的 stale-term 与架构图文本扫描

## Known Issues

- `runtime-web` 仍定位为可选调试壳，不是正式产品主入口
- 历史 `platform-web` / `platform-api` 仍保留在兼容或归档路径，后续仅用于迁移对照，不再作为默认主线
- 完整演示仍依赖本地环境变量、账号与下游服务可用性

## Recommended Demo Flow

建议对外演示按下面顺序展开：

1. `overview`
2. `assistants`
3. `chat`
4. `sql-agent`
5. `testcase`
6. `threads`
7. `runtime` / `audit` / `users`
