# v0.1.2 - Agent 工作台可演示版

> 正式发布说明草案，可直接用于 GitHub Release。

## Summary

`v0.1.2` 是 `v0.1.1` 之后的一轮功能补强与体验收口版本。
这一版把项目级知识工作区正式拉通到 `platform-web-vue -> platform-api-v2` 控制面链路，补齐 `testcase v2` 的运行时与前端入口，并继续压缩 `chat` / `testcase` 工作区的界面噪音，同时把 AI 执行系统的仓库级标准、用法指南和叶子路由文档固定下来。

## Highlights

- `platform-web-vue` 新增项目级知识工作区主线，补齐 `documents / retrieval / graph / settings` 页面、上传对话框、查询设置和图谱交互
- `platform-api-v2` 新增项目知识控制面能力，承接项目知识路由、权限、上游代理与契约校验
- `runtime-service` 新增 `test_case_service_v2` 主线与相关知识/文档持久化能力，补强 testcase 新一代执行路径
- `chat` / `testcase` 工作区继续收口为更紧凑的对话导向体验，补齐 compact / focus 模式下的细节表现
- 仓库补齐 AI 执行系统 current-standard、使用指南和 leaf resolver 文档，正式固定 AI 任务 intake / routing / verification 路径

## Added

- 新增项目级知识工作区能力，包括知识文档、检索、图谱、设置与上传主链路
- 新增 `platform-api-v2` 的项目知识 facade、服务层、SQLAlchemy repository 与相关集成测试
- 新增 `runtime-service` 的 `test_case_service_v2` graph、文档持久化、知识查询守卫与相关技能文档
- 新增仓库级 AI 执行系统标准、使用指南、leaf resolver 与 `.omx/specs` helper artifacts

## Changed

- 收紧 `chat` / `sql-agent` / `testcase` 工作区的界面骨架，让目标上下文、对话列表、输入区与消息区更聚焦主会话
- 统一知识工作区回到共享前端壳和列表页范式，减少滚动裁切、页面结构漂移和上传交互分散问题
- 调整 README、部署/开发文档与知识设计文档，让入口路径、发布路径和 AI 执行路由更清晰
- 让共享多模态解析默认值支持环境配置，同时保留显式覆盖与运行时覆盖路径

## Fixed

- 修复 testcase 文档详情面板在旧数据形态下无法正常预览与原始下载的问题
- 修复项目知识设置页尾部斜杠请求导致的代理跳转与鉴权丢失问题
- 修复 compact / focus 模式下 chat 全局状态条、跟随滚动与局部布局噪音问题
- 修复若干知识页面滚动裁切、工作区内容溢出与上传元数据校验体验问题

## Known Issues

- `runtime-web` 仍定位为可选调试壳，不是正式产品主入口
- 项目级知识完整演示仍依赖本地 LightRAG / 上游知识服务、账号与环境变量可用
- `testcase v2` 已具备主线入口，但更大范围的端到端回归仍建议在完整 demo 环境下再走一轮

## Recommended Demo Flow

建议对外演示按下面顺序展开：

1. `overview`
2. `projects`
3. `knowledge`（documents / retrieval / graph / settings）
4. `chat`
5. `sql-agent`
6. `testcase`
7. `threads`
8. `runtime` / `audit` / `users`
